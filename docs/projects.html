The Data Job Market — Skills & Pay Dashboard

I built an end‑to‑end analysis of Tech hiring landscape, unifying postings from multiple sources into a single, interactive Tableau view. The workflow covers web scraping (LinkedIn with Selenium) and the Adzuna API, an ETL pipeline into MySQL/DuckDB, and modeling with dbt using a medallion architecture (Bronze → Silver → Gold).

  • Standardized salary normalization (hourly/monthly → annual CAD) and robust deduplication.
  • Built a skill taxonomy with regex to tag tools (SQL, Python, Tableau, Azure, AWS, Databricks, dbt).
  • Calculated KPIs: median salary by role & city, remote share, and AI/Cloud mentions trend.
  • Published an interactive Tableau dashboard with filters for role, location, seniority, and skills.

Result: a recruiter‑ready view of which roles pay best, which skills are in demand, and where opportunities cluster across Canada.

Learnings

  • Developed an in-depth understanding of how job market data reflects skill demand and salary trends across tech roles.
  • Enhanced proficiency in end-to-end data engineering — from scraping raw data to modeling and visualizing insights.
  • Gained experience building automated ETL pipelines and modular transformations using dbt and DuckDB.
  • Learned to design Tableau dashboards that blend technical accuracy with clear storytelling for decision-makers.
  • Strengthened problem-solving through debugging data models, schema design, and metadata documentation.
Electric Vehicle Market Analysis — Tableau & Python

This project analyzes the growth of Electric Vehicles (EVs) versus Plug-in Hybrid Electric Vehicles (PHEVs) using real-world sales and pricing data. The goal was to uncover adoption trends, price segments, and manufacturer competitiveness across global markets.

  • Cleaned and merged datasets in Python using Pandas for accurate KPI computation.
  • Created interactive Tableau dashboards highlighting EV vs PHEV market share, MSRP distribution, and YoY growth.
  • Derived insights into how Tesla compares against other manufacturers and identified regions with the fastest EV adoption rates.

Learnings

  • Learned to structure a data analytics workflow combining Python preprocessing with Tableau visualization.
  • Strengthened understanding of KPI design and data storytelling for sustainability-driven industries.
  • Gained hands-on experience transforming messy datasets into clean, actionable insights.
  • Developed a deeper appreciation for the power of visual analytics in communicating environmental impact.
(Currently Working on this!) Apple Retail Sales — dbt + DuckDB Medallion

I’m building a modern analytics stack with dbt on DuckDB, organized as a medallion architecture (Bronze → Silver → Gold). Raw CSVs are staged via sources.yml, cleaned into staging models (naming: stg_*), and transformed to dim/fact tables for the final analytics layer.

  • Data quality with tests (unique, not_null, accepted_values) and source freshness.
  • Performance using incremental models and surrogate keys via macros.
  • Metrics in Gold: Revenue, Average Order Value (AOV), Units/Txn, WoW/ MoM growth, and product- & city-level breakdowns.
  • Documentation with dbt docs (lineage graph) and exposures for downstream dashboards.

Outcome: a repeatable, version-controlled pipeline that turns raw retail files into trustworthy, dashboard-ready tables.

Expected Learnings

  • Mastering advanced dbt concepts such as incremental models, seeds, and reusable macros.
  • Understanding medallion architecture at a deeper level by managing dependencies across layers.
  • Improving data governance through tests, documentation, and exposure tracking.
  • Learning to optimize performance and scalability of analytics workflows using DuckDB.
  • Strengthening storytelling by connecting dbt outputs to visual dashboards in Tableau or Power BI.
